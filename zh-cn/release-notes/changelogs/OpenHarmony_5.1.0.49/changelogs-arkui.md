# ArkUI子系统ChangeLog
## cl.arkui.1 半模态跟手样式弹窗显示位置避让规则变更
**访问级别**

公开接口

**变更原因**

基础能力增强，半模态跟手样式弹窗支持开发者自定义弹出方向，并根据位置避让流程决定弹窗最终显示位置

**变更影响**

该变更为不兼容变更。

变更前行为：半模态跟手样式弹窗默认弹出位置为绑定组件底部，会根据剩余空间情况同绑定组件左边和右边作对齐布局，同时可将半模态弹窗布局至窗口外，不做任何半模态面板高度的压缩

变更后行为：开发者可在bindOptions中的placement(设置弹窗默认弹出位置，默认值Bottom) 和 placementOnTarget(所有位置均放不下时，是否允许弹窗覆盖在绑定节点上)两个字段，自定义半模态跟手样式弹窗相关的弹出位置信息。若开发者指定位置放不下半模态弹窗，则根据UX避让规则，遵循"下->上->右->左"、"右->左->下->上"等的先做垂直方向翻转、再做水平90°方向尝试的避让流程，寻找合适的弹出位置。如果所有位置均无法容纳跟手样式弹窗，则根据placementOnTarget字段，若为True，则在开发者指定弹出位置覆盖绑定节点显示；若为false，则寻找四个方向中，宽度能够容纳半模态弹窗宽度且高度值为最大的位置，在压缩半模态面板高度后显示


**起始API Level**

11

**变更发生版本**

从OpenHarmony SDK 5.1.0.49 版本开始。

**变更的接口/组件**

bindSheet
SheetType.POPUP

**适配指导**

默认行为变更，开发者无需适配，需注意若api16前现实的效果为底部左对齐或者底部右对齐，可以通过设置placement值为BottomLeft或BottomRight来做对应效果显示
